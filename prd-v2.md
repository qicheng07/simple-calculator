## 一、产品概述

### 1.1 产品定位
从单一计算器升级为**多工具聚合的生活效率工作台**，整合计算器、待办清单等常用工具，提供统一、高效的使用体验。

### 1.2 目标用户
| 用户群体 | 使用场景 | 核心诉求 |
|---------|---------|---------|
| 学生群体 | 作业计算、任务管理 | 工具切换便捷、数据持久化 |
| 上班族 | 日常计算、工作待办 | 界面统一、操作高效 |
| 家庭用户 | 账单计算、生活规划 | 简单易用、功能实用 |

### 1.3 核心价值
- **多工具聚合**：一站式获取常用工具，无需打开多个页面
- **无缝切换**：标签页切换，保留各工具状态
- **数据持久化**：本地存储，刷新不丢失数据
- **统一体验**：视觉规范一致，降低学习成本

## 二、功能定义

### 2.1 功能清单

| 功能模块 | 优先级 | 功能描述 |
|---------|--------|---------|
| 统一工作台架构 | P0 | 顶部标签页、工具容器、数据隔离 |
| 待办清单工具 | P0 | 增删改查、完成状态、本地存储 |
| 视觉规范统一 | P0 | 配色方案、组件规范、响应式设计 |
| 动效交互 | P1 | 标签切换动效、操作反馈动效 |
| 进阶功能 - 汇率换算 | P1 | 实时汇率（模拟）、多币种转换 |

### 2.2 功能详细说明

#### 2.2.1 统一工作台架构（P0）

**功能描述**：
- 顶部固定标签栏，包含"计算器"、"待办清单"、"汇率换算"
- 点击标签切换对应工具，保留各工具内部状态
- 各工具数据完全隔离，互不影响

**标签页规则**：
- 默认激活"计算器"标签
- 激活标签有高亮显示
- 标签宽度自适应，支持响应式布局

**数据隔离机制**：
- 各工具使用独立的状态管理
- 本地存储按工具前缀隔离（如 `todo_xxx`、`calc_xxx`）

**页面布局**：
```
┌─────────────────────────────────────┐
│  [计算器] [待办清单] [汇率换算]      │ ← 顶部标签栏
├─────────────────────────────────────┤
│                                     │
│         工具内容区域                 │
│                                     │
└─────────────────────────────────────┘
```

#### 2.2.2 待办清单工具（P0）

**功能描述**：
- 新增待办：输入文本添加新任务
- 删除待办：点击删除按钮移除任务
- 编辑待办：双击任务文本进入编辑模式
- 完成状态：点击复选框标记完成/未完成
- 数据持久化：使用 localStorage 存储

**待办数据结构**：
```javascript
{
  id: string,        // 唯一标识（时间戳）
  text: string,      // 待办文本
  completed: boolean, // 完成状态
  createdAt: number,  // 创建时间戳
  updatedAt: number   // 更新时间戳
}
```

**交互规则**：
1. **新增待办**
   - 输入框位于顶部，支持回车提交
   - 空输入不允许提交
   - 提交后清空输入框，自动聚焦

2. **删除待办**
   - 每个待办右侧有删除按钮
   - 点击删除显示确认动画
   - 删除后更新统计

3. **编辑待办**
   - 双击待办文本进入编辑模式
   - 编辑模式显示输入框，支持回车保存、ESC取消
   - 失去焦点自动保存

4. **完成状态**
   - 左侧复选框，勾选后文本显示删除线并变灰
   - 完成的任务可以继续编辑或删除

5. **统计展示**
   - 底部显示：已完成 / 总数
   - 提供"清除已完成"按钮

**界面布局**：
```
┌─────────────────────────┐
│ [ 输入新待办... ] [+]   │ ← 输入区域
├─────────────────────────┤
│ ☑  完成的任务   [×]     │
│ ☐  未完成的任务 [×]     │
│ ☑  另一个完成的 [×]     │
├─────────────────────────┤
│ 已完成: 2 / 3  [清除已完成] │ ← 统计区域
└─────────────────────────┘
```

#### 2.2.3 视觉规范统一（P0）

**配色方案**：
| 用途 | 颜色值 | 说明 |
|------|--------|------|
| 主色 | #FF9500 | 橙色（沿用计算器） |
| 背景 | #F5F5F7 | 浅灰背景 |
| 卡片背景 | #FFFFFF | 白色卡片 |
| 文字主色 | #1D1D1F | 深灰文字 |
| 文字辅助 | #86868B | 浅灰文字 |
| 成功 | #34C759 | 绿色 |
| 危险 | #FF3B30 | 红色 |
| 边框 | #E5E5EA | 边框线 |

**组件规范**：
- 按钮：圆角 8px，高度 44px，字体 16px
- 输入框：圆角 8px，高度 44px，边框 1px
- 标签页：圆角 8px，内边距 12px 20px
- 卡片：圆角 16px，阴影 0 2px 12px rgba(0,0,0,0.08)

**字体规范**：
- 标题：20px，medium
- 正文：16px，regular
- 辅助：14px，regular
- 小字：12px，regular

#### 2.2.4 动效交互（P1）

**1. 标签切换动效**
- 点击标签时，下划线从当前标签滑动到目标标签
- 内容区域使用淡入淡出过渡（opacity 0→1，duration 200ms）
- 切换时保持标签栏稳定

**2. 操作反馈动效**
- 按钮点击：缩放 0.95→1，duration 150ms
- 待办完成：复选框勾选动画 + 文本颜色渐变
- 删除待办：项高度收缩 + 淡出，duration 300ms
- 新增待办：项从上方滑入，duration 300ms

#### 2.2.5 进阶功能 - 汇率换算（P1）

**功能描述**：
- 支持多种常见货币转换
- 模拟实时汇率（内置常用汇率）
- 双向输入转换

**支持币种**：
- CNY 人民币
- USD 美元
- EUR 欧元
- JPY 日元
- GBP 英镑
- HKD 港币

**交互规则**：
- 左侧输入框：源货币金额 + 币种选择
- 右侧输入框：目标货币金额 + 币种选择
- 任意一侧输入，另一侧自动计算
- 点击交换按钮可互换源和目标

**汇率数据（模拟）**：
```javascript
{
  "CNY": 1,
  "USD": 0.14,
  "EUR": 0.13,
  "JPY": 21.0,
  "GBP": 0.11,
  "HKD": 1.09
}
```

## 四、界面设计要求

```

│  生活效率工具箱                     │ ← 页面标题

```

### 4.2 响应式断点

| 设备类型 | 断点宽度 | 布局调整 |
|---------|---------|---------|
| 移动端 | < 480px | 标签垂直排列 |
| 平板 | 480px - 768px | 标签水平排列 |
| 桌面 | > 768px | 标签水平排列，内容区域居中 |

### 4.3 工具容器规范
- 最大宽度：480px
- 水平居中
- 卡片样式，带阴影
- 内边距：24px

## 五、技术说明

### 5.1 技术栈
- 前端：HTML5 + CSS3 + JavaScript (原生)
- 存储：localStorage
- 部署：GitHub Pages

### 5.2 数据存储方案

**localStorage 键命名规范**：
- `workbench_activeTab` - 当前激活标签
- `todo_items` - 待办清单数组
- `calc_state` - 计算器状态（可选）
- `exchange_rates` - 汇率数据

### 5.3 模块架构

```
workbench/
├── index.html          # 主页面
├── styles.css          # 统一样式
├── script.js           # 主入口
├── modules/
│   ├── tab-manager.js  # 标签管理
│   ├── calculator.js   # 计算器模块
│   ├── todo.js         # 待办模块
│   └── exchange.js     # 汇率模块
└── utils/
    ├── storage.js      # 存储工具
    └── animation.js    # 动画工具
```
